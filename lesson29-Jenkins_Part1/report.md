# Отчет: Jenkins - Part1

# Сравнительный анализ CI/CD систем

| Критерий | Jenkins | GitLab CI/CD | CircleCI | Travis CI |
| :--- | :--- | :--- | :--- | :--- |
| **Тип платформы** | Self-hosted (свой сервер) | Облачный (SaaS) или Self-hosted | Преимущественно облачный (SaaS) | Облачный (SaaS) |
| **Цена и лицензия** | Бесплатно (Open Source) | Есть бесплатный тариф / Подписка | Есть бесплатный тариф / Оплата за ресурсы | Только платные тарифы |
| **Формат конфигурации** | Jenkinsfile (Groovy / Declarative) | `.gitlab-ci.yml` (YAML) | `.circleci/config.yml` (YAML) | `.travis.yml` (YAML) |
| **Гибкость настройки** | Максимальная (через плагины) | Высокая | Высокая (шаблоны Orbs) | Средняя / Базовая |
| **Масштабируемость** | Ручная настройка нод и агентов | Легкая (через GitLab Runner) | Автоматическая (в облаке) | Ограничена тарифом |
| **Экосистема плагинов** | Самая большая (тысячи плагинов) | Много встроенных инструментов | Средняя | Небольшая |
| **Сложность поддержки** | Высокая (требует администрирования) | Низкая для SaaS / Средняя для Self-hosted | Минимальная | Минимальная |
| **Главное преимущество** | Полная независимость и кастомизация | Все инструменты DevOps в одном месте | Высокая скорость сборки и кэширования | Простая интеграция с GitHub |
| **Главный недостаток** | Устаревший интерфейс, сложные обновления | Тяжеловесный интерфейс платформы | Дорого при больших нагрузках | Нет полноценного бесплатного тарифа |
